# Style Guide — Survey / Review Papers

## Taxonomy Section
- Use Mermaid `mindmap` for hierarchical taxonomies.
- Use Mermaid `timeline` for chronological development.
- A single paper may have multiple diagrams for different facets.

## Key Approaches Section
- Group methods by category under `###` subsections.
- Per-method format: **Key insight / Advantage / Limitation** as bullet points.
- Use comparison tables when 3+ methods share the same category.

## Comparison Section
- Main table: Methods / Metrics / Year. Mark SOTA in bold.
- Include qualitative trade-off table (Aspect × Method matrix).
- Sort by year or performance; state the sorting rule.

## Open Challenges Section
- Each challenge: **bold heading** + description + why it is difficult.
- End with numbered future directions.

## Reference Handling
- Group references concisely: "Prior works [1,2,3] propose…" not sequential [1]…[2]…[3].
